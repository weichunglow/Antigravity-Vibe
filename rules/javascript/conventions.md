# 💻 Javascript & Code Structure

This document outlines standard conventions for Javascript and code structure.

---

## 💻 Javascript & Code Structure

1.  **Semantic HTML holds logic**: Always structure components using semantic HTML `<header>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>`.
2.  **Single Responsibility Principle**: Functions should do one thing and do it well. Avoid large monolithic functions.
3.  **Naming Conventions**:
    -   Files: `camelCase` or `kebab-case`.
    -   Functional elements: Class names starting with uppercase if they represent a component, otherwise standard `kebab-case`.
    -   IDs: Unique, descriptive IDs for easy browser testing.
