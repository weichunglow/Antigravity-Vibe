# 🧹 General Rules & Housekeeping

This document outlines general repo cleanliness and security guidelines for the workspace.

---

## 🧹 Housekeeping & Cleanup

1.  **No Dead Code**: Commented out blocks of code should be removed rather than committed.
2.  **Standard Formatting**: Use Prettier if available. Set tabs/spaces to consistent values (2 spaces preferred).
3.  **Documentation Guidelines**: Every script must have a top-level comment explaining its purpose and inputs/outputs.

---

> [!CAUTION]
> Never commit passwords or unencrypted secrets to the repository. Use `.env.example` templates if required.
